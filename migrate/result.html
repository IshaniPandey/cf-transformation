Testing your docker image

You can test your docker image by running it locally. This local testing may generate errors as there is no backend services available.

  1. Generate your docker image
    cd :tgtdir.
    docker build -t :applname. .
  2. Run your docker image
    docker run --rm --name testcon :applname.
    You may have to map application ports to expose them so that you can access your application using -p port:port
  3. When you are done testing, you can remove your running docker using the command:
    docker stop testcon

Deploying application to :tgttype.

Perform the following actions:

  1. Go to the conversion path directory cd :tgtdir.
  2. Generate the docker image:docker build -t :applname. .
  3. Tag the docker image for a docker repository that is accessible to the cluster.docker tag :applname. <registry>/<namespace>/:applname.
  4. Store the docker image into a docker repository that is accessible to the cluster.docker push <registry>/<namespace>/:applname. :deployapp.
